# chiaki-sys

Raw FFI bindings to [chiaki-ng](https://github.com/streetpea/chiaki-ng), auto-generated by [bindgen](https://github.com/rust-lang/rust-bindgen).

This crate provides the low-level C API surface of the chiaki-ng core library. For a safe, idiomatic Rust API, use the [`chiaki`](https://crates.io/crates/chiaki) crate instead.

---

## Build

The build script (`build.rs`) automatically:

1. Compiles the chiaki-ng C core library via CMake (GUI, CLI, tests, and Steam Deck native all disabled)
2. Generates `bindings.rs` via bindgen from all headers in `lib/include/chiaki/`
3. Emits `cargo:rustc-link-*` directives for all transitive static dependencies

On Windows, the build script uses **gcc + Ninja** via MSYS2/MinGW-w64 to compile
the C library. Run `cargo build` from within an **MSYS2 MinGW 64-bit** shell after
installing dependencies — see the `chiaki` crate README for setup instructions.

All C library dependencies must be installed before running `cargo build`.
See the [`chiaki` crate README](https://github.com/streetpea/chiaki-ng/tree/main/bindings/rust/chiaki#building) for per-platform installation instructions (Linux, macOS, Windows).

---

## Versioning

The crate version tracks the chiaki-ng release (e.g. `1.9.9`). Regenerate bindings after a C API change by running:

```bash
cargo build -p chiaki-sys
```

Do **not** edit `src/lib.rs` manually — it is overwritten on each build.
